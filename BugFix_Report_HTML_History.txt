Laki Web IDE - HTML Tag & History Persistence Fix Report

ISSUE SUMMARY:
Users were losing HTML tag assignments and visual designer history because the data was stored in volatile memory or hidden system folders that were not included in project-specific backups.

FIXES IMPLEMENTED:

1. Memory Corruption Fix (ViewBean.java):
   - Changed the copy() method to perform a DEEP COPY of the parentAttributes map.
   - Benefit: Prevents undo/redo history snapshots from sharing the same data as the live view, which was causing history corruption.

2. Permanent Storage Redirection (WebProjectSyncManager.java):
   - Modified syncFile() to save the current visual designer state (ArrayList<ViewBean>) into the project's own folder.
   - Path: [ProjectRoot]/settings/[PageName]_htmltags.json
   - Benefit: This file is now part of the project structure and will be automatically included in standard backups.

3. Path Management Unification (LakiFiles.java & wq.java):
   - Migrated legacy system paths from wq.java to LakiFiles.java.
   - Ensured that "complex" and "simple" project roots are handled consistently.
   - Benefit: Fixes "File Not Found" errors when the Backup system looks for projects in different locations than the Editor.
